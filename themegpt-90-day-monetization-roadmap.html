Operating Rules

Keep the roadmap skeleton, but run lean. Ship only what changes revenue, reduces funnel friction, or protects trust.
  • One source of truth for offers: checkout behavior, pricing page, extension copy, and legal text must match.
  • Privacy consistency is mandatory: either stay no-tracking end-to-end or add explicit consent + updated policy.
  • Use one launch channel at a time to keep signal clean and execution realistic for a solo builder.
  • No paid acquisition until an organic conversion baseline is proven.

Phase 1 Goal

Remove contradictions before launch so every customer sees the same offer and the same trust story.

Current mismatch to resolve first: code applies a 30-day trial to monthly, while roadmap copy previously positioned trial on yearly.

Week 1 - Canonical Offer Decision

  • Choose one final trial design and keep it everywhere (code, web copy, extension, support docs, legal pages).
  • Confirm lifetime and early-adopter rules in plain language and in Stripe metadata handling.
  • Create one pricing source file/document and link all customer-facing copy to it.

Week 2 - Live Payment QA + Trust Consistency

  • Run live purchase matrix: monthly, yearly, single theme, cancel, refund, and re-login entitlement recovery.
  • Verify webhooks and DB writes for checkout completion, renewal, cancellation, and trial-end notifications.
  • Align privacy wording with actual measurement plan before launch messaging.
  • Publish final Terms and Privacy updates if wording changed.

Phase 2 Goal

Convert existing demand first, run one launch channel, and collect high-signal feedback.

Weeks 3-4 - In-Extension Monetization Messaging

  • Tighten locked-theme upgrade messaging and CTA wording in popup and pricing destination.
  • Add one lifecycle nudge after repeated premium-theme attempts (helpful, not spammy).
  • Ensure account and purchase state messaging is clear for trialing, active, canceled, and lifetime users.
  • Launch non-incentivized review ask after a positive usage moment (for example after 3+ successful theme applies).

Weeks 5-6 - One Launch Channel (Recommended: Product Hunt)

  • Prepare one launch asset pack: before/after GIFs, 30-second demo, short founder story, clear CTA.
  • Run only one external launch channel for a full 2-week window to keep attribution clean.
  • Use weekly scorecard to identify the biggest funnel drop and fix only that one bottleneck each week.

Phase 3 Goal

Scale what already converts, then run limited paid tests with strict kill rules.

Weeks 7-9 - Scale Proven Organic Motion

  • Double down only on the best-performing channel from Phase 2.
  • Release 1-2 new premium themes to re-engage users and create fresh launch moments.
  • Run a focused win-back campaign to trial drop-offs with a clear single CTA.

Weeks 10-12 - Paid Test (Only if Gate Passed)

  • Start paid only if organic conversion from install to paid is at least 3% for 2 consecutive weeks.
  • Run one paid channel at low spend first ($150-300 total test budget).
  • Kill paid test if CAC is greater than $12 or payback exceeds 2 months.
  • If paid test passes, scale gradually and keep weekly holdout comparison.
