## Trigger Condition

- `v2.2.2` status changes from **Pending review** to **Approved/Published** in Chrome Web Store dashboard.
- Do not submit `v2.3.0` before this gate clears.

---

## Preflight Checklist

- Confirm `main` contains commit `c1fffae` and tag `v2.3.0`.
- Confirm GitHub Actions secret `SUBMIT_KEYS` is present and valid.
- Confirm no hotfix branch is waiting that should be included in the same submission.
- Confirm local status docs still state: "`v2.3.0` queued pending `v2.2.2` clearance."

Quick verification commands:

```bash
git rev-parse --short main
git tag --list "v2.3.0"
gh secret list --repo <owner>/themegpt-v2.0 | rg SUBMIT_KEYS
```

---

## Submission Steps

1. Open GitHub Actions in repo `themegpt-v2.0`.
2. Select workflow **Submit Extension to Web Store**.
3. Click **Run workflow**.
4. Choose branch `main`.
5. Start run and monitor to completion.

CLI alternative:

```bash
gh workflow run submit-extension.yml --ref main --repo <owner>/themegpt-v2.0
gh run list --workflow submit-extension.yml --repo <owner>/themegpt-v2.0 --limit 1
```

Expected workflow stages:

1. Checkout repository
2. Install dependencies
3. Build shared package (`packages/shared`)
4. Build extension (`apps/extension`)
5. Package zip (`apps/extension/build/chrome-mv3-prod.zip`)
6. Publish using `PlasmoHQ/bpp` with `SUBMIT_KEYS`

---

## Success Criteria

- GitHub workflow run status = **Success**
- Publish step (`Browser Platform Publish`) = **Passed**
- Chrome Web Store dashboard shows `v2.3.0` submitted for review

## Failure Handling

- Build/package failure: fix code on `main`, rerun workflow.
- `SUBMIT_KEYS`/publish failure: rotate credential bundle, update GitHub secret, rerun.
- CWS API acceptance failure: verify package metadata/version in extension build output and retry.
