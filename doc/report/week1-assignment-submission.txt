CRITERIA 2: CONTENT MANAGEMENT SYSTEM (CMS) - 3 CONTENT TYPES
================================================================================

STATUS: COMPLETE (Firebase Firestore Implementation)

ThemeGPT implements structured content types via TypeScript interfaces with
Firestore as the persistence layer.

CONTENT TYPE 1: Themes
-----------------------
Location: packages/shared/src/index.ts (Lines 20-33)

interface Theme {
  id: string;
  name: string;
  category: 'christmas' | 'core' | 'premium';
  colors: {
    '--cgpt-bg': string;
    '--cgpt-surface': string;
    '--cgpt-text': string;
    '--cgpt-text-muted': string;
    '--cgpt-border': string;
    '--cgpt-accent': string;
  };
  isPremium: boolean;
}

- 12 themes defined (3 free, 9 premium)
- Categorized by type (christmas, core, premium)
- Color system with CSS variable mapping


CONTENT TYPE 2: License Entitlements
-------------------------------------
Location: packages/shared/src/index.ts (Lines 210-227)

interface LicenseEntitlement {
  active: boolean;
  type: 'subscription' | 'lifetime';
  maxSlots: number;
  permanentlyUnlocked: string[];
  activeSlotThemes: string[];
}

- Stored in Firestore 'licenses' collection
- CRUD operations via apps/web/lib/db.ts
- API endpoints: /api/checkout, /api/verify, /api/sync


CONTENT TYPE 3: Token Statistics
---------------------------------
Location: packages/shared/src/index.ts (Lines 13-18)

interface TokenStats {
  user: number;
  assistant: number;
  total: number;
  lastUpdated: number;
}

- Tracks ChatGPT token usage
- Stored locally in browser (privacy-first)
- Real-time updates via content script


FIRESTORE CONFIGURATION:
- Project: themegpt-website
- Rules: firestore.rules (public read/write until 2026-01-10)
- Admin SDK: apps/web/lib/firebase-admin.ts
- Client SDK: apps/web/lib/firebase.ts
